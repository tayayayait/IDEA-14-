# PHASE 4 - 리포트 문구 정리 (2026-04-25)

## 목적
- Step6 리포트에서 가독성이 낮은 문구를 정리하고, 영문 상태값/HTML 엔티티 노출을 제거한다.

## 변경 파일
- `src/pages/Step6Report.tsx`

## 반영 내용

1. 리포트 텍스트 정규화 함수 추가
- `normalizeReportText` / `decodeHtmlEntities`를 추가해 표시 텍스트를 정규화.
- `&rsquo;`, `&lsquo;`, `&middot;`, `&#39;` 같은 HTML 엔티티가 그대로 노출되는 문제를 완화.

2. 리포트 데이터 표시 문구 정리
- 국가명/요약/리스크/인증·규제 텍스트를 정규화 후 렌더링.
- AI 요약 fallback 문구를 명확한 한국어로 정리.
- 기본 실행 과제 문구를 간결하게 수정.

3. 출처 표 문구 정리
- API 명칭을 리포트용 한국어 라벨로 표시.
- 상태값을 `idle/success/empty/error` 대신 한국어 라벨로 변환:
  - 대기 / 정상 / 조회 결과 없음 / 오류 / 부분 반영 등
- 오류코드 없는 경우 `없음`으로 표시.

## 기대 효과
- 리포트 본문과 출처 섹션의 문구가 한국어 중심으로 통일된다.
- 국가명/본문에 HTML 엔티티가 그대로 보이는 현상이 줄어든다.
- 출처 상태 해석이 쉬워진다.
