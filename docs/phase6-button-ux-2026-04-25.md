# Phase 6: 버튼 UX 개선 (2026-04-25)

## 목표
- 주요 액션 버튼의 중복 클릭을 줄이고, 실행 중 상태를 명확히 표시한다.
- 키보드 사용 시 카드형 버튼의 포커스 가시성을 확보한다.
- 단계 이동 버튼을 현재 실행 상태와 연동해 잘못된 타이밍 이동을 방지한다.

## 반영 파일
- `src/pages/Projects.tsx`
- `src/pages/Step3Countries.tsx`
- `src/pages/Step4CountryDetail.tsx`
- `src/pages/Step5Safety.tsx`
- `src/pages/Step6Report.tsx`

## 변경 내용
1. 프로젝트 생성 버튼 UX
- 상단 `새 분석 시작`, 빈 상태 `지금 시작하기` 버튼에 `creating` 연동.
- 생성 중 스피너/라벨(`분석 생성 중...`) 표시 및 중복 클릭 방지.

2. 카드형 버튼 접근성/포커스 개선
- 프로젝트 카드, Step3 후보국 카드의 `button`에 `type="button"` 명시.
- `focus-visible` 링 스타일을 추가해 키보드 탐색 시 현재 포커스가 보이도록 개선.

3. Step3 추천 실행 버튼 상태 개선
- 실행 중 라벨을 `추천 실행 중...`으로 변경.
- 실행 중에는 `안전성 검토` 이동 버튼도 비활성화해 상태 전이를 단순화.

4. Step4 상세 분석/과제 생성 버튼 상태 개선
- `taskGenerating` 상태 추가.
- `과제 생성` 버튼에 실행 중 스피너/라벨(`과제 생성 중...`) 및 비활성화 조건 추가.
- `상세 분석 실행` 버튼도 과제 생성 중에는 비활성화하고 실행 중 라벨 표기.

5. Step5 리포트 이동 버튼 가드 강화
- `reportNavigating` 상태 추가.
- 검토 실행 중이거나 검토 근거가 없는 경우 리포트 이동 차단 + 안내 토스트 제공.
- 리포트 이동 중 스피너/라벨(`리포트 이동 중...`) 표시.
- `검토 실행` 버튼도 리포트 이동 중 비활성화.

6. Step6 액션 버튼 상호 잠금
- `AI 요약 생성`과 `PDF 다운로드`를 상호 배타적으로 비활성화.
- 각 버튼에 실행 중 라벨(`AI 요약 생성 중...`, `PDF 생성 중...`) 적용.
