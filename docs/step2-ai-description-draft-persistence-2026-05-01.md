# Step2 AI 설명 초안 유지 수정 (2026-05-01)

## 원인
- `AI 설명 초안 작성` 버튼으로 받은 설명은 화면 상태만 갱신하고 `project_products`에 즉시 저장하지 않았다.
- 사용자가 저장 후 다음 단계로 이동하거나 새로고침하기 전에 화면을 벗어나면 생성된 문구가 DB에 남지 않아 다시 빈 값으로 보일 수 있었다.
- 초기 제품 로드 응답이 늦게 도착하는 경우, 이미 생성된 설명을 기존 빈 설명으로 덮어쓸 수 있었다.

## 변경
- 버튼 생성 성공 시 기존 `project_products` 행을 즉시 업데이트한다.
- 기존 제품 행이 없으면 현재 사용자 기준으로 `project_products` 행을 새로 생성해 설명 초안을 저장한다.
- 사용자가 설명을 입력했거나 AI 초안이 반영된 뒤에는 늦게 도착한 초기 로드 응답이 설명 값을 덮어쓰지 않도록 보호한다.
