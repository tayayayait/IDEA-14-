# Phase 0 HS/HSK 추천 즉시 차단 보강 (2026-04-27)

## 배경
- `DRAM/NAND Flash` 입력에서도 제조장비 계열(예: `8479903000`)이 자동 선택되는 오분류가 재현됨.
- 원인:
  - 제품명/설명/맥락 토큰을 동일 가중치로 점수화해 맥락 잡음이 과반영됨.
  - 메모리 반도체 의도에 대한 챕터 가드레일이 약함.
  - AI 재정렬 후 점수 재정렬이 다시 적용되어 순서 보정 효과가 약해짐.

## 적용 변경
- 파일: `supabase/functions/ai-hs-suggest/index.ts`
  - 입력 토큰을 소스별 가중치로 분리:
    - 제품명 1.0, 부품 0.95, 모델 0.75, 설명 0.45, 목표시장메모 0.35
  - 메모리 반도체 의도(`dram/sram/nand/flash/memory/메모리` 등) 탐지 시:
    - 메모리 직접 근거/854232 챕터 가점
    - 제조장비형 후보(84류+장비 키워드) 감점
  - `flash` 단독 키워드 오탐(예: flashbulb) 방지:
    - 메모리 의도 판별에 강한 토큰(`dram/sram/nand/memory/메모리`) 컨텍스트를 함께 요구
    - 행 매칭도 `flash memory/플래시 메모리` 중심으로 제한
  - AI 재정렬 결과가 있을 때는 해당 순서를 유지하고, 없을 때만 점수 정렬 수행

- 파일: `src/lib/hs-candidate-relevance.ts`
  - 메모리 반도체 질의에서 비메모리 후보를 하드 필터로 차단
  - 필터 이후 비어 있으면 메모리 후보(`854232` 또는 메모리 토큰 매칭) 우선 fallback

- 파일: `src/test/hs-candidate-relevance.test.ts`
  - 메모리 질의에서 제조장비 후보가 제외되고 `8542321030`이 유지되는 케이스 추가
  - `flashbulb` 기계류가 NAND Flash 메모리 후보로 오인되지 않는 케이스 추가
  - `flash eliminator` 비메모리 후보가 제외되는 케이스 추가

## 기대 효과
- `DRAM/NAND Flash`와 같은 메모리 반도체 입력에서 제조장비/무관 품목의 상위 자동선택 확률 감소.
- 프론트/서버 양쪽에서 동일 방향(의도 기반)으로 방어해 단일 실패 지점 축소.
