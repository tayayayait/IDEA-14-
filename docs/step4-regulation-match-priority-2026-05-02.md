# Step4 수입규제 매칭 우선순위

적용 범위: `country-detail`의 KOTRA 수입규제 데이터.

## 확정 규제

확정 규제는 다음 조건을 모두 만족해야 한다.

- 수입국 ISO2 코드가 선택 국가와 일치
- 조사/규제 대상국이 한국 또는 전세계 대상
- HS/HSK 기준이 1순위 또는 2순위

우선순위:

1. 국가 + HS/HSK 정확 일치
2. 국가 + HS 6자리 일치

## 검토 필요 후보

아래 조건은 확정 규제로 계산하지 않고 `review_required`로만 저장한다.

3. 국가 + HS 4자리 일치 + 제품명 강한 유사도
4. 국가 + 제품명 강한 유사도

제품명 유사도는 제품명/검증된 제품 토큰 중심으로 판단한다. 소재, 구성품, 일반 품목어 단독 매칭은 후보 생성에 사용하지 않는다. 예: `aluminum`, `sheet`, `plastic`, `baby`, `infant`, `부품`, `소재`, `제품`, `시트`, `유아` 단독 매칭은 수입규제 후보로 승격하지 않는다.

## 저장 메타데이터

`project_regulations.raw`에 다음 값을 저장한다.

- `match_priority`: 1~4순위
- `matched_tokens`: 제품명 유사도에 사용된 토큰
- `hs_match_level`: `hsk_exact`, `hs_exact`, `hs6_prefix`, `hs4_prefix`, `none`
- `hs_partial_match`: 3순위 여부

## 한계

이 정책은 KOTRA 수입규제 데이터의 매칭 정확도를 높이는 정책이다. CPSC, ASTM 같은 제품안전/기술규제 전체를 대체하지 않는다. 해당 유형은 별도 데이터 소스와 별도 카테고리로 분리해야 한다.
