# Step4 KOTRA 국가정보 표시 정책 (2026-05-04)

## 목적

KOTRA 국가정보 API의 긴 원문을 Step4 기본 화면에 그대로 노출하지 않는다. 기본 화면은 제품명, HS, HSK, 문단 키워드 기반으로 선별한 짧은 완결 요약만 표시하고, 원문은 `자세히 보기` 접힘 영역에 보존한다.

## 관련성 분류

`src/lib/kotra-national-info.ts`는 문단별로 다음 4단계를 판정한다.

- `direct`: 제품명 토큰, HS/HSK, HS 4자리, HS 2자리 품목군이 문단과 명확히 맞는 경우.
- `conditional`: 중고 장비, 전기전자제품, RoHS, 특정 조건 적용 표현처럼 현재 품목군과 조건부로 연결되는 경우.
- `common`: 모든 수입품, 수입통관, HS 코드 확인, 관세율 확인, FTA 원산지증명서 등 공통 적용 정보.
- `unrelated`: 현재 품목군과 다른 자동차, 농식품, 비료, 식품, 의료기기, 완구 등 품목별 규제.

HS Chapter 매핑은 1차 품목군 추정용이다. 최종 표시는 제품명 토큰, HS/HSK 신호, 문단 내 품목명과 규제대상 키워드를 함께 비교한다. 특정 제품명 기준 예외 `if`는 추가하지 않는다.

## 기본 화면 표시

- 직접 관련 문단은 `직접 관련`에 표시한다.
- 조건부 문단은 `조건부 확인`에 표시한다.
- 공통 문단은 `공통 참고`에 표시한다.
- 비관련 문단은 기본 화면에 표시하지 않는다.
- 직접 관련 문단이 없으면 “현재 원문에서 해당 HS코드 또는 제품명과 직접 일치하는 규제는 확인되지 않았습니다.”를 표시한다.
- 경제 동향, 경제 전망은 국가 공통 정보로 취급해 관련성 필터 없이 문장 단위 요약만 표시한다.

## 문장 요약 기준

기본 화면 요약은 글자 수로 자르지 않는다.

- 문장 단위로만 표시한다.
- `7.85%`, `7.09%`, `6.5~7.0%` 같은 숫자 소수점은 문장 종료로 보지 않는다.
- `2025년 성장률은 7.`처럼 숫자 뒤에서 끊긴 문장은 표시하지 않는다.
- 너무 긴 문장은 괄호나 출처구를 제거해 완결 문장으로 줄일 수 있을 때만 표시하고, 아니면 제외한다.
- 카드별 기본 요약은 최대 2~3개 bullet을 우선한다.

## 검증

주요 검증은 `src/test/kotra-national-info.test.ts`와 `src/test/step4-source-cards-ui.test.ts`에서 수행한다.

- 반도체/HS 845611 문맥에서 자동차, 농산물, 비료, 식품, 의료기기, 완구 문단이 비관련으로 분류되는지 확인한다.
- HS 87, 01~24, 30, 33, 39, 72~83, 90의 1차 품목군 판정을 확인한다.
- 공통 참고와 조건부 확인 문단이 직접 관련과 섞이지 않는지 확인한다.
- 경제 수치 문장이 소수점에서 잘리지 않는지 확인한다.
- Step4가 `buildNationalInfoPresentation` 결과를 사용하고 원문은 `presentation.rawText` 기반 접힘 영역에 보관하는지 확인한다.
