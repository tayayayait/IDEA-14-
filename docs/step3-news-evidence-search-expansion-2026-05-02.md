# Step3 뉴스 근거 검색 범위 확장 (2026-05-02)

## 변경 목적

후보국별 뉴스 근거가 부족하게 저장되는 경우를 줄이기 위해 `recommend-country-news`의 KOTRA 뉴스 검색 범위와 후보 풀을 확장했다. 기존 뉴스 수집, 제품/국가 관련성 필터, AI/규칙 분류, `news_scope` 저장 구조는 유지했다.

## 변경 내용

- 제품 뉴스 검색 한도를 `12개 쿼리 x 20건`에서 `16개 쿼리 x 30건`으로 확대했다.
- 국가 뉴스 검색 한도를 `14개 쿼리 x 10건`에서 `20개 쿼리 x 20건`으로 확대했다.
- AI 검토 후보 수를 `30건`에서 `60건`으로 확대했다.
- 저장 후보 선택 한도를 기본 카테고리당 `4건`에서 `6건`으로, 국가 배경 fallback은 `3건`에서 `5건`으로 확대했다.
- 제품 검색어는 기존 제품명, 강한 토큰, HS 설명, 제품군 힌트, HS6 기준을 유지하면서 강한 토큰/HS 설명/제품군 힌트 범위를 늘리고 HS4+제품 토큰 조합을 추가했다.
- 국가 검색어는 단일 대표 국가명 중심에서 국가 alias 최대 3개를 사용하도록 확장했다. 각 alias에 제품 키워드, HS6, 주요 수출환경 키워드를 조합한다.
- `api_call_logs.detail`에 제품/국가 쿼리 수, 수집 후보 수, AI 검토 한도, 저장 후보 한도를 기록하도록 했다.

## 유지한 필터

- 제품 직접 관련성 판정은 `assessNewsRelevance`를 그대로 사용한다.
- 국가 일치 판정은 `assessCountryNewsMatch`를 그대로 사용한다.
- 제품 수출 근거로 사용할 수 있는지에 대한 최종 방어 필터는 `hasDefensibleProductExportFit`를 그대로 사용한다.
- 프론트엔드 호출 방식, Supabase 함수명, DB schema, `rationale.sources` 필드 구조는 변경하지 않았다.
