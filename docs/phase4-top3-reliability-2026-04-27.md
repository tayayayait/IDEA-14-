# Phase 4 - 추천 국가 Top 3 신뢰성 보강 (2026-04-27)

## 목적
- 추천 국가 점수와 근거 연결의 불일치를 줄이고, Step 3/4 사용자 흐름에서 상위 1순위 국가 기준이 일관되게 유지되도록 보강한다.

## 변경 파일
- `src/pages/Step3Countries.tsx`
- `supabase/functions/recommend-countries/index.ts`
- `supabase/functions/_shared/recommendation.ts`

## 핵심 변경
1. Top 3 재추천 후 상세 링크 동기화
- 추천 재실행 직후 Step 3 목록을 다시 로드하고, 1순위 국가 코드를 선택 상태/저장 상태로 즉시 갱신.
- 후속 Step 4 상세 이동 시 이전 국가가 아닌 최신 1순위 국가를 기준으로 이동.

2. 국가 매칭 뉴스 오탐 억제
- 제품 뉴스의 국가 코드는 `natn/regn` 메타데이터를 우선 사용.
- 메타데이터 부재 시 제목/키워드만 제한적으로 사용하고, 본문 기반 국가 추정은 배제.
- 국가 뉴스 매칭도 동일하게 메타데이터 우선, 제목/키워드 보조 규칙으로 정리.

3. 제품 뉴스 근거 vs 국가 뉴스 근거 분리 표시
- Step 3 근거 목록 라벨을 `제품 뉴스 근거` / `국가 뉴스 근거`로 분리.
- 표시는 제품 근거를 우선 노출하고, 국가 배경 근거는 후순위로 정렬.

4. 목표시장 메모 지역어 감지 보강
- 목표시장 메모에서 `동남아`, `북미` 계열 표현을 지역 코드 세트로 확장.
- 동남아: `VN/TH/MY/ID`, 북미: `US/MX`.

5. 인증/규제 0점 국가 고득점 완화
- fallback 점수에서 인증/규제 신호가 없으면 결제·안전 점수도 보수적으로 계산.
- 최종 총점 계산 시 인증/규제 0점 조합에 대해 상한/감점 규칙 적용.

## 테스트 보강
- 목표시장 메모 지역어(동남아/북미) 확장 파싱 테스트 추가.
- 인증/규제 신호 유무에 따라 결제/안전 점수가 보수적으로 변화하는지 테스트 추가.
- Step 3 근거 라벨 분리(`제품 뉴스 근거`/`국가 뉴스 근거`) 기대값 반영.
