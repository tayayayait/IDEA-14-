# AI 산단수출 코파일럿 상세서

산업단지 제조기업의 수출 준비 과정을 공공데이터 기반 AI 워크플로로 통합하는 웹 서비스의 구현 기준 문서.

## 0. 문서 기준

| 항목 | 내용 |
|---|---|
| 문서 목적 | 개발자와 디자이너가 별도 해석 없이 화면, UX, 컴포넌트, 상태, 예외 처리를 구현하기 위한 기준 |
| 적용 범위 | MVP 기준 웹 앱 |
| 사용 API 범위 | P0 필수 API 10개만 사용 |
| 최종 확인일 | 2026-04-22 |
| 제외 범위 | 결제, 회원 과금, 실제 전략물자 최종 판정, 실제 인증 적합성 최종 판정, 개별 바이어 신용평가 |

## 1. 제품 정의

### 1.1 서비스 개요

| 항목 | 내용 |

| 프로젝트명 | AI 산단수출 코파일럿 |

| 핵심 사용자 | 산업단지 입주 중소·중견 제조기업, 수출 초보기업, 지자체·산단 기업지원 담당자, 무역 컨설턴트 |

## 2. 정보 구조

### 2.1 최상위 내비게이션

| 메뉴 | 경로 | 목적 |
|---|---|---|
| 새 분석 | `/projects/new` | 기업·제품 기반 신규 수출 분석 시작 |

### 2.2 분석 단계

| 단계 | 이름 | 필수 완료 조건 | 다음 단계 이동 조건 |
|---:|---|---|---|
| 1 | 기업 확인 | 공장 또는 기업 1개 선택 | 회사명, 생산품, 업종 중 2개 이상 확보 |
| 2 | 품목 확인 | 제품명과 HS/HSK 후보 입력 | 사용자가 분석 기준 품목을 확정 |

| 4 | 인증·규제 | 후보국별 인증·수입규제 조회 | 최소 1개 국가 상세 확인 가능 |
| 5 | 리스크 분석 | 국가·업종·결제 위험 조회 | 위험도 라벨 산출 |
| 6 | 안전·통제 | 전략물자 플래그와 제품안전 유사사례 조회 | 검토 결과 표시 |
| 7 | 리포트 | 요약, 근거, 다음 실행 과제 생성 | PDF 미리보기 가능 |

### 3.2 화면 1: 기업·공장 검색

| 항목 | 명세 |
|---|---|
| 경로 | `/projects/new/company` |
| 목적 | 수출 분석 대상 제조기업과 공장 정보 확정 |
| API | 한국산업단지공단_공장등록생산정보조회서비스 |

화면 구조:

| 구역 | 구성 |
|---|---|
| 헤더 | 제목 `분석할 기업 또는 공장을 찾으세요`, 보조문 `회사명, 공장명, 생산품 중 하나 이상을 입력합니다.` |
| 검색 폼 | 회사명 입력, 공장명 입력, 지역 선택, 생산품 키워드 입력, 검색 버튼 |
| 검색 결과 | 표 형태. 컬럼: 회사명, 공장명, 지역, 업종, 생산품, 선택 버튼 |
| 선택 요약 | 선택된 기업의 주소, 생산품, 업종, 출처, 조회일 표시 |
| 빈 상태 | `검색 결과 없음`, 조건 초기화 버튼, 수동 입력 버튼 |

검색 폼 규칙:

| 필드 | 타입 | 필수 | 제한 |
|---|---|---|---|
| 회사명 | 텍스트 | 조건부 | 2자 이상 |
| 공장명 | 텍스트 | 조건부 | 2자 이상 |
| 지역 | 콤보박스 | 선택 | 시·도 단위 |
| 생산품명 | 텍스트 | 조건부 | 2자 이상 |

검증 규칙:

- 회사명, 공장명, 생산품명 중 하나 이상 입력해야 검색 버튼 활성화.
- 검색 중 버튼 문구는 `검색 중`으로 변경하고 버튼 비활성화.
- API 오류 시 결과 영역에 오류 패널 표시.

### 3.3 화면 2: 제품·품목 프로파일

| 항목 | 명세 |
|---|---|
| 경로 | `/projects/{projectId}/product` |
| 목적 | 분석 기준 제품과 HS/HSK 후보 확정 |
| API | 무역안보관리원_HSK 연계표 정보 |

화면 구조:

| 구역 | 구성 |
|---|---|
| 기업 요약 바 | 선택 기업명, 공장 주소, 대표 생산품, 업종 |
| 제품 입력 | 제품명, 제품 설명, 주요 부품, 모델명, 목표 시장 메모 |
| HS/HSK 후보 | 후보 코드 입력 테이블, 코드 설명, 사용 여부 토글 |
| AI 후보 제안 | 제품명·생산품 기반 후보 제안. 근거가 없으면 표시하지 않음 |
| 확정 액션 | `분석 기준 품목으로 확정` 버튼 |

입력 규칙:

| 필드 | 타입 | 필수 | 제한 |
|---|---|---|---|
| 제품명 | 텍스트 | 필수 | 2~80자 |
| 제품 설명 | 텍스트 영역 | 필수 | 20~1000자 |
| 주요 부품 | 태그 입력 | 선택 | 최대 20개, 태그당 30자 |
| 모델명 | 텍스트 | 선택 | 80자 이하 |
| HS/HSK 후보 | 코드 입력 | 필수 | 숫자 6~10자리 |

처리 원칙:

- HS/HSK 자동 확정 금지.
- AI는 후보와 근거만 제시한다.
- 사용자가 `분석 기준 품목`을 선택해야 다음 단계 이동 가능.

### 3.4 화면 3: 수출 후보국 추천 대시보드

| 항목 | 명세 |
|---|---|
| 경로 | `/projects/{projectId}/countries` |
| 목적 | 후보국 Top 3와 추천 근거 비교 |
| API | KOTRA_국가정보, KOTRA_해외시장뉴스 |

화면 구조:

| 구역 | 구성 |
|---|---|
| 추천 요약 | Top 3 국가 카드, 추천 점수, 주요 근거 3개 |
| 비교 차트 | 국가별 시장성, 규제부담, 결제위험, 인증난이도, 전략물자주의 점수 |
| 국가 비교 표 | 국가명, 시장 신호, 인증 필요, 수입규제, 결제위험, 종합 점수 |
| 뉴스 근거 | 국가별 관련 해외시장뉴스 목록. 제목, 국가, 게시일, 원문 링크 |

추천 점수 규칙:

```text
종합점수 = 시장성 30 + 인증용이성 20 + 규제위험 20 + 결제안정성 20 + 제품안전/통제주의 10
```

점수 표시 규칙:

| 점수 | 라벨 | 색상 |
|---:|---|---|
| 80~100 | 우선 검토 | Success |
| 60~79 | 검토 가능 | Info |
| 40~59 | 주의 필요 | Warning |
| 0~39 | 고위험 | Danger |
| 데이터 부족 | 확실한 정보 없음 | Neutral |

### 3.5 화면 4: 국가별 상세 리스크

| 항목 | 명세 |
|---|---|
| 경로 | `/projects/{projectId}/countries/{countryCode}` |
| 목적 | 특정 국가의 인증, 수입규제, 결제위험을 상세 확인 |
| API | KOTRA_해외인증정보, KOTRA_수입규제품목 정보, K-SURE 3종 API |

화면 구조:

| 구역 | 구성 |
|---|---|
| 국가 헤더 | 국가명, 추천 순위, 종합 점수, 마지막 조회일 |
| 인증 패널 | 인증명, 적용 품목, 필요서류, 절차, 유효기간, 원문 링크 |
| 수입규제 패널 | HS 코드, 규제유형, 규제내용, 시행일, 근거 링크 |
| 국가·업종 리스크 | 신용등급, 업종 위험지수, 위험도 라벨 |
| 결제 리스크 | 평균 결제기간, 연체율, 권장 결제조건, 주의 메시지 |
| 다음 실행 과제 | `인증 확인`, `규제 원문 검토`, `무역보험 상담`, `기관 확인` 체크리스트 |

표시 조건:

- 인증정보가 없으면 `확실한 정보 없음`으로 표시하고 빈 값을 추정하지 않는다.
- 수입규제 정보가 없으면 `조회된 규제 없음`과 `데이터 기준일`을 함께 표시한다.
- K-SURE 정보는 개별 바이어 판단이 아니라 국가·업종 단위 참고자료로 표시한다.

### 3.6 화면 5: 전략물자·제품안전 검토

| 항목 | 명세 |
|---|---|
| 경로 | `/projects/{projectId}/safety` |
| 목적 | 전략물자 가능성과 제품안전·리콜 유사사례 확인 |
| API | 무역안보관리원_HSK 연계표 정보, 국가기술표준원/SafetyKorea_제품 안전인증 및 리콜 정보 |

화면 구조:

| 구역 | 구성 |
|---|---|

| 제품안전 검색 결과 | 제품명·모델명 기준 인증/리콜 유사사례 목록 |
| 리스크 안내 | 자동 판정 아님, 기관 확인 필요 문구 |
| 조치 체크리스트 | 전략물자 자가판정, 인증번호 확인, 리콜 유사사례 원문 확인 |

전략물자 상태:

| 상태 | 표시 문구 | 처리 |
|---|---|---|
| `clear` | 현재 입력 코드 기준 연계 항목 없음 | Neutral |
| `review_required` | 전략물자 검토 필요 | Warning |
| `unknown` | 코드 미확정 또는 데이터 부족 | Neutral |
| `error` | HSK 데이터 조회 실패 | Danger |

제품안전 상태:

| 상태 | 표시 문구 | 처리 |
|---|---|---|
| `no_match` | 유사 인증·리콜 사례 없음 | Neutral |
| `similar_found` | 유사 사례 확인 | Info |
| `recall_found` | 리콜 유사사례 확인 | Danger |
| `unknown` | 확실한 정보 없음 | Neutral |

## 4. 사용자 흐름

### 4.1 표준 분석 흐름

```text
1. 사용자가 회사명 또는 공장명을 입력한다.
2. 시스템이 공장등록생산정보를 조회한다.
3. 사용자가 기업·공장을 선택한다.
4. 사용자가 제품명, 제품 설명, HS/HSK 후보를 입력한다.
5. 사용자가 분석 기준 품목을 확정한다.
6. 시스템이 KOTRA 국가정보와 해외시장뉴스를 조회한다.
7. 시스템이 추천 수출국 Top 3를 생성한다.
8. 시스템이 국가별 해외인증과 수입규제를 조회한다.
9. 시스템이 K-SURE 국가·업종·결제위험을 조회한다.
10. 시스템이 HSK 연계표와 제품안전·리콜 데이터를 조회한다.
11. 사용자가 국가별 상세 근거를 확인한다.
12. 사용자가 PDF 실행 리포트를 생성한다.
```

### 4.2 공장 검색 실패 흐름

```text
1. 검색 결과가 없다.
2. 시스템은 검색 조건 초기화와 수동 입력을 제안한다.
3. 사용자가 수동 입력을 선택하면 회사명, 지역, 업종, 생산품을 직접 입력한다.
4. 수동 입력 데이터에는 `사용자 입력` 배지를 표시한다.
5. 수동 입력 데이터는 공공데이터 근거 점수에 반영하지 않는다.
```

### 4.3 일부 API 실패 흐름

```text
1. 국가 추천에 필요한 일부 API가 실패한다.
2. 성공한 API 결과로 1차 분석을 표시한다.
3. 실패한 영역에는 `조회 실패`, `다시 시도`, `마지막 성공 데이터 없음`을 표시한다.
4. 종합 점수에는 `부분 산출` 배지를 표시한다.
5. PDF 리포트에는 실패 API와 미확인 항목을 명시한다.
```

| 429 | `요청 한도를 초과했습니다. 잠시 후 다시 시도하세요.` | 60초 후 재시도 버튼 활성화 |
| 500 | `제공기관 API 응답이 불안정합니다.` | 1회 자동 재시도 후 실패 처리 |
| Timeout | `응답 시간이 초과되었습니다.` | 부분 결과 표시 |
| 결과 없음 | `조회된 데이터가 없습니다.` | 빈 상태 표시, 수동 입력 또는 조건 변경 제안 |

### 11.2 데이터 불일치

| 상황 | 처리 |
|---|---|
| 국가명이 API별로 다름 | 내부 국가코드 매핑 테이블 사용. 매핑 실패 시 사용자 선택 |
| 업종 코드가 없음 | 업종명 텍스트 기반으로 표시하되 점수 계산에는 낮은 신뢰도 적용 |
| HS/HSK 코드 미확정 | 인증·규제·전략물자 단계 진입 제한 |
| 제품안전 유사 검색 과다 | 상위 10개만 표시하고 `유사도 낮음` 라벨 제공 |
| KOTRA 뉴스 근거 부족 | 시장성 점수에서 뉴스 항목 제외, `확실한 정보 없음` 표시 |

### 11.3 AI 응답 예외

| 상황 | 처리 |
|---|---|
| 근거 없음 | `확실한 정보 없음` 반환 |
| 출처 없는 문장 생성 | 화면에 표시하지 않음 |
| 법적 판정처럼 표현 | `기관 확인 필요` 문구로 대체 |
| 상충 데이터 존재 | 두 출처를 모두 표시하고 `출처 간 차이 있음` 경고 |
